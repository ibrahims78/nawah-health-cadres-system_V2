# خطة تحسينات منصة مسارات

## المجموعة A — الأمان وجودة الكود 🔴

### A1 — Zod Validation على الـ API
**المشكلة:** عدة routes تقبل بيانات بدون التحقق منها بـ schema محدد.
**القرار:** ✅ إضافة Zod validation على كل routes الإضافة والتعديل.

### A2 — حماية تسريب الحقول الحساسة
**المشكلة:** حذف `googleServiceAccountKeyEnc` و`telegramBotTokenEnc` يدوياً — إذا أُضيف حقل جديد قد يُنسى.
**القرار:** ✅ استخدام Drizzle select بحقول آمنة محددة مسبقاً بدلاً من الحذف اليدوي.

## المجموعة B — اتساق الواجهة 🟡

### B1 — استبدال browser confirm() بـ Dialog احترافي
**المشكلة:** ProjectRecords و GlobalSettings تستخدم `window.confirm()` القديم بدلاً من الـ Dialog الاحترافي المستخدم في بقية الصفحات.
**القرار:** _(في انتظار الإجابة)_

### B2 — نظام Toast موحّد
**المشكلة:** كل صفحة تعرض نتائج العمليات بـ ResultBox مضمّن يُزيح المحتوى. يوجد Toast في المشروع لكنه غير مُفعَّل بشكل موحّد.
**القرار:** _(في انتظار الإجابة)_

### B3 — Skeleton Loading بدلاً من Spinner
**المشكلة:** كل الصفحات تعرض Loader2 مركزي واحد — يسبب layout shift مزعج.
**القرار:** _(في انتظار الإجابة)_

## المجموعة D — ميزات محرر النماذج 🔴

### D1 — Conditional Fields (حقول مشروطة)
**المشكلة:** لا يمكن إظهار/إخفاء حقل بناءً على إجابة حقل آخر (مثال: "إذا اخترت نعم، حدد النوع").
**القرار:** _(في انتظار الإجابة)_

### D2 — Field Validation Rules (قواعد التحقق على الحقول)
**المشكلة:** لا يمكن للمسؤول تحديد قواعد مثل: حد أدنى/أقصى لعدد الأحرف، أو نمط regex معين.
**القرار:** _(في انتظار الإجابة)_

### D3 — File Upload كنوع حقل جديد
**المشكلة:** نوع الحقول المتاحة لا يشمل رفع ملف أو صورة.
**القرار:** _(في انتظار الإجابة)_

### D4 — Drag-and-drop لإعادة ترتيب الحقول
**المشكلة:** ترتيب الحقول يتم بتعديل رقم يدوياً — لا توجد واجهة سحب وإفلات مرئية.
**القرار:** _(في انتظار الإجابة)_
